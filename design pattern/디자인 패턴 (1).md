# 디자인 패턴 (1)

## 디자인 패턴의 소개

**디자인 패턴**: 프로그램을 설계할 때 발생했던 문제점들을 객체간의 상호관계 등을 이용하여 해결할 수 있도록 하나의 규약 형태로 만들어 놓은 것.

**디자인 패턴의 종류**
  - 생성 패턴: 객체 생성 방법이 들어간 디자인 패턴 (싱글톤, 팩토리 패턴)
  - 구조 패턴: 큰 구조를 만들 때 유연하고 효율적으로 만드는 방법이 들어간 디자인 패턴 (프록시 패턴)
  - 행동 패턴: 객체나 클래스 간의 알고리즘, 책임 할당에 관한 디자인 패턴 (이터레이터, 옵저버, 전략 패턴)

## 라이브러리와 프레임워크의 차이

공통점: 사용될 수 있는 특정한 기능들을 모듈화한 것
차이점: 라이브러리는 폴더명, 파일명 등에 대한 규칙이 없고 프레임워크에 비해 자유로우며, 프레임워크는 라이브러리에 비해 좀 더 엄격하다.

## 싱글톤 패턴

**싱글톤 패턴**: 하나의 클래스에 오직 하나의 인스턴스만 가지는 패턴. 인스턴스 생성을 효율적으로 하는 장점이 있지만 의존성이 높아지고 TDD(Test Driven Development)를 할 때 불편하다는 단점이 있다.

예제 코드
```java
class Singleton {
    private static class singleInstanceHolder {
        private static final Singleton INSTANCE = new Singleton();
    }
    public static Singleton getInstance() {
        return singleInstanceHolder.INSTANCE;
    }
}
public class HelloWorld {
    public static void main(String []args) {
        Singleton a = Singleton.getInstance();
        Singleton b = Singleton.getInstance();
        System.out.println(a.hashCode());
        System.out.println(b.hashCode());
        if (a == b) {
            System.out.println(true);
        }
    }
}
```

## 싱글톤 패턴을 구현하는 7가지 방법

1. 단순한 메서드 호출
2. synchronized
  - 인스턴스를 반환하기 전까지 격리 공간에 놓기 위해 synchronized 키워드로 잠금을 할 수 있다. 최초로 접근한 스레드가 해당 메서드 호출시에 다른 스레드가 접근하지 못하도록 잠금을 걸어준다. 이 때 getInstance() 메서드를 호출할 때마다 lock이 걸려 성능저하가 된다.
  예제 코드
  ```java
  public class Singleton {
    private static Singleton instance;

    private Singleton() {
    }

    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
  }
  ```
3. 정적 멤버(즉시 초기화, Eager Initialization)
  - 정적(static) 멤버로 싱글톤을 만드는 방법. Singleton 클래스가 참조될 때 로딩과 초기화가 이루어지는데 이 때 메모리 상에서 한 번만 생성되고 프로그램 종료시까지 유지된다.
4. 정적 블록(즉시 초기화, Eager Initialization)
5. 정적 멤버와 Lazy Holder(중첩 클래스)
  - singleInstanceHolder라는 내부 클래스를 하나 더 만듦으로써, Singleton 클래스가 최초에 로딩되더라도 함께 초기화되지 않고, getInstance()가 호출될 때 singleInstanceHolder 클래스가 로딩되어 인스턴스를 생성하게 된다.
6. 이중 확인 잠금(DCL-Double Checked Locking)
  - 인스턴스 생성 여부를 싱글톤 패턴 잠금 전에 한 번, 객체를 생성하기 전에 한 번 총 2번 체크하면 인스턴스가 존재하지 않을 때만 잠금을 걸 수 있기 때문에 앞서 생겼던 문제점을 해결할 수 있다.
  예제 코드
  ```java
  public class Singleton {
    private volatile Singleton instance;

    private Singleton() {
    }

    public Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
  }
  ```
  **volatile**: Java에서는 thread 2개가 열리면 변수를 메인 메모리 (RAM)으로부터 가져오는 것이 아니라 캐시 메모리에서 각각의 캐시 메모리를 기반으로 가져오게 된다. 그로 인해 변수 값 불일치 문제가 발생할 수 있고, volatile 키워드를 추가하면 메인 메모리를 기반으로 저장하고 읽어오기 때문에 이 문제를 해결할 수 있다.
7. enum
  - enum의 인스턴스는 기본적으로 thread safe*한 점이 보장된다
  - thread safe*: 멀티스레드 프로그래밍 환경에서 여러 스레드가 동시에 공유 자원(변수, 객체, 함수 등)에 접근해도 프로그램의 실행 정합성이 보장되며, 예기치 않은 동작이나 데이터 손상 없이 정상적으로 작동하는 상태 

## 팩토리 패턴

**팩토리 패턴**: 상속 관계에 있는 두 클래스에서 상위 클래스가 중요한 뼈대를 결정하고, 하위 클래스에서 객체 생성에 관한 구체적인 내용을 결정하는 패턴. 유연성과 유지 보수성이 증가된다는 장점이 있다.

예제 코드
```java
enum CoffeeType {
    LATTE,
    ESPRESSO
}
abstract class Coffee {
    protected String name;

    public String getName() {
        return name;
    }
}
class Latte extends Coffee {
    public Latte() {
        name = "Latte"
    }
}
class Espresso extends Coffee {
    public Espresso() {
        name = "Espresso"
    }
}
class CoffeeFactory {
    public static Coffee createCoffee(CoffeeType type) {
        switch (type) {
            case LATTE:
                return new Latte();
            case ESPRESSO:
                return new Espresso();
            default:
                throw new IllegalArgumentException("Invalid coffee type: " + type);
        }
    }
}
public class Main {
    public static void main(String[] args) {
        Coffee coffee = CoffeeFactory.createCoffee(CoffeeType.LATTE);
        System.out.println(coffee.getName());
    }
}
```

## 이터레이터 패턴

**이터레이터 패턴**: 이터레이터를 사용하여 컨테이너*의 요소들에 접근하는 디자인 패턴. 각기 다른 자료구조들을 똑같은 인터페이스로 순회를 쉽게 할 수 있다는 장점이 있음. 
컨테이너*: 동일한 요소들을 담아놓는 집합 (배열, 맵)

예제 코드
```js
const mp = new Map()
mp.set('a', 1)
mp.set('b', 2)
mp.set('ccc', 3)
const st = new Set()
st.add(1)
st.add(2)
st.add(3)
const a = []
for(let i = 0; i < 10; i++)a.push(i)

for(let aa of a) console.log(aa)
for(let a of mp) console.log(a)
for(let a of st) console.log(a)
```
