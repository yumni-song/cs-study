# 네트워크의 기초

## 1. 네트워크, 처리량, 트래픽, 대역폭, RTT
### 네트워크
노드와 링크가 서로 연결되어 있으며, 리소스를 공유하는 집합
- 노드: 서버, 라우터, 스위치 등 네트워크 장치
- 링크: 유선/무선과 같은 연결매체

### 처리량
링크내에서 성공적으로 전달된 데이터의 양, 얼만큼의 트래픽을 처리했는지 나타냄

### 트래픽
특정 시점에 링크 내에 흐르는 데이터의 양
- 트래픽이 많아졌다. = 흐르는 데이터가 많아졌다.
- 처리량이 많아졌다. = 처리되는 트래픽이 많아졌다.

### 대역폭
주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 최대 비트 수, 최대로 처리할 수 있는 트래픽

### RTT(Round Trip Time: 왕복 지연 시간)
신호를 전송하고 해당 신호의 수신 확인에 걸린 시간을 더한 값, 어떤 메시지가 두 장치 사이를 왕복하는 데 걸린 시간

## 2. 네트워크 토폴로지: 버스, 스타, 트리, 링, 메시
### 네트워크 토폴로지
노드와 링크가 어떤 구조로 연결되었는지 나타내는 방식

### 버스 토폴로지
회선 하나에 여러개의 노드가 연결된 구조
- 장점: 설치 비용이 적고, 유지 보수가 쉽다. 한 노드에 장애가 발생해도 전체 네트워크에 영향 주지 않음.
- 단점: 회선이 망가지면 전체 네트워크에 장애가 발생, 회선에 트래픽이 생기면 정체 현상 발생 가능성이 높음, 모든 노드가 동일 회선을 공유하므로 패킷 도청(스니핑)이나 위조(스푸핑) 등의 보안 위협에 취약

### 스타 토폴로지
하나의 중앙 노드를 중심으로 여러개의 말단 노드가 별 모양처럼 각각 독립적으로 연결된 구조로, 모든 통신은 항상 중앙 노드를 통해서만 이루어진다.
- 장점: 보안 관리 용이성(보안 장비를 중앙에 집중 적용 가능)
- 단점: 중앙 노드 에러시 큰 문제 발생 

### 트리 토폴로지
계층적 구조를 가진 네트워크 형태로, 트리 구조처럼 상위 노드와 하위 노드가 단계적으로 연결됨.
노드 추가 및 삭제는 리프 노드에서 용이하나, 중간 계층 이상의 노드는 영향도가 크기 때문에 구조 변경이 복잡할 수 있다.
- 장점: 리프노드의 확장이 용이하며, 리프노드의 에러는 나머지 부분에 영향을 미치지 않는다.
- 단점: 특정 노드 트래픽 집중시 하위 모든 노드에 병목 현상 발생 가능성이 있으며, 루트 노드에 문제가 생기면 전체 네트워크에 큰 문제가 발생한다.

### 링형 토폴로지
노드들이 고리 형태로 연결된 네트워크 구조
- 장점: 네트워크 상에는 단 하나의 토큰이 존재하고 토큰을 소유한 노드만 데이터 송신, 수신 권한을 가진다. 이 때문에 다른 노드가 동시에 데이터를 전송할 일이 전혀 없으므로 데이터 충돌이 없다.
- 단점: 데이터 전송시 모든 노드를 거쳐야 하므로 전송 지연 발생 가능

### 메시 토폴로지
모든 노드들이 직접 서로 연결되는 그물망 형태의 네트워크 구조
- 장점: 안정성이 높고 트래픽 분산 가능
- 단점: 비용이 많고, 노드 추가/삭제가 매우 어려움

## 3. 병목 현상과 네트워크 토폴로지의 필요성
### 병목 현상
네트워크에서 트래픽이 특정 구간에 집중되어 데이터 흐름 속도가 제한되는 상황
- 원인: 대역폭 부족, 서버/애플리케이션 처리 능력 한계, 네트워크 토폴로지 문제

### 토폴로지의 필요성
토폴로지를 파악함으로써 병목 현상을 해결하는 척도가 된다.
- 해결 사례: 구축된 시스템의 토폴로지를 알고 있다면 어떠한 부분에 어떠한 회선 또는 어떠한 서버의 용량을 증가시켜야 하는지 알 수 있다.

## 4. 유니캐스트, 멀티캐스트, 브로드캐스트
### 유니캐스트
1:1 통신. 가장 일반적인 네트워크 전송 형태 (예: HTTP 통신)

### 멀티 캐스트
1:N 통신. 연결된 모든 노드들에게 데이터를 전달하지는 않고 특정 그룹에게만 데이터 전달

### 브로드캐스트
1:N 통신. 그룹이 아닌 연결되어 있는 모든 노드에게 데이터를 전달. (예: ARP)

## 5. 네트워크의 분류: LAN, MAN, WAN
### LAN(Local Area Network: 근거리 통신망)
MAN, WAN보다 높은 안정성, 속도를 가지며 소규모네트워크(보통 허브나 스위치로 연결된 네트워크)이다.
하나의 논리적 주소인 IP를 기반으로 여러개의 물리적 주소인 MAC 주소로 구별하는 네트워크

### MAN(Metropolitan Area Network: 대도시 통신망)
도시와 도시의 통신망으로, 두개 이상의 LAN이 연결되어 구성된다.

### WAN(Wide Area Netwrk: 광역 통신망)
국가와 국가와의 통신망을 뜻하며 인터넷이라고도 한다.
