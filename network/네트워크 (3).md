# IP 주소 체계

## 1. IP 주소, MAC 주소, ARP와 RARP
### IP 주소(Internet Protocol address)
논리적 주소로, 컴퓨터 네트워크에서 장치들이 서로를 인식하고 통신을 하기 위해서 사용하는 특수한 번호이며 IP를 기반으로 통신한다고도 하지만 사실상 그 밑에 물리적 주소인 MAC 주소를 통해 통신한다.

### MAC 주소 (Media Access Control Address)
네트워크 인터페이스에 할당된 고유 식별자이며, 보통 장치의 NIC(Network Interface Card)에 할당된다.
48비트로 이루어졌으며, 24비트의 OUI와 24비트의 UAA로 이루어져있다.
- OUI: IEEE에서 할당한 제조사 코드
- UAA: 제조사에서 구별되는 코드

### ARP와 RARP
ARP를 통해 논리적 주소인 IP 주소를 물리적 주소인 MAC 주소로 변환하며, RARP를 통해 물리적 주소인 MAC 주소를 논리적 주소인 IP 주소로 변환하기도 한다.

### ARP의 과정
1. 해당 IP 주소에 맞는 MAC 주소를 찾기 위해 해당 데이터를 "브로드캐스팅"을 통해 연결된 네트워크에 있는 장치한테 모두 보낸다.
2. 맞는 장치가 있다면 해당 장치는 보낸 장치에게 유니캐스트로 데이터를 전달해 주소를 찾게 된다.

## 2. IPv4와 IPv6
### IPv4
32비트로 표현되는 주소체계이며 2^32개의 주소(41억 9천만)를 표현할 수 있다. 8비트 단위로 점을 찍어 4개로 구분해서 표현하며 보통 8비트를 10진수로 표현해서 말한다. 이 주소체계만으로는 부족하기 때문에 NAT, 서브네팅 등 여러개의 부수적인 기술이 생겨났다.

### IPv6
128비트로 표현되는 주소체계이며 2^128개의 주소를 표현한다. 16비트씩 8개로 구분하고 16비트는 16진수로 변환되어 콜론으로 구분하여 표시하며 앞의 연속되는 0은 생략 가능하다. (앞의 64비트는 네트워크 주소, 뒤 64비트는 인터페이스 주소(호스트 주소)로 이루어져 있다.)

- IPv6에는 데이터 통신을 암호화할 수 있는 IPSec이 내장되어 있다.
- IPv6는 IPv4 헤더의 불필요한 필드를 제거하여 보다 빠른 처리가 가능하다.
- IPv4는 체크섬이 있지만 IPv6는 체크섬이 없다. IPv4는 헤더 체크섬 필드를 통해, 헤더의 비트가 손상되었는지 여부를 1의 보수합 방식으로 검증해 손상된 패킷을 확인하고 폐기한다. IPv6의 헤더 체크섬 필드가 삭제된 이유는 이미 상위 프로토콜인 TCP와 UDP의 헤더에서 체크섬을 통해 패킷의 무결성을 보장하고 있기 때문이다.

## 3. 클래스풀(Classful IP Addressing)
네트워크 주소를 매기고 그에 따라 네트워크의 크기를 다르게 구분하여 클래스를 할당하는 주소체계. 구분하는 기준자(1, 2, 3 옥텟)를 서브넷마스크라고 한다.

IP 주소는 인터넷 주소로, 네트워크 주소, 호스트 주소 두 부분으로 나뉜다.
- 네트워크 주소가 동일 -> 로컬 네트워크
- 호스트 주소 = 호스트를 구분하기 위한 주소
- 네트워크 호스트는 컴퓨터 네트워크에 연결된 컴퓨터나 기타 장치

### 클래스 A
- 2^24-2 = 한 네트워크당 호스트 ID 약 1600만개
- 네트워크 주소 범위: 1~126으로 시작 (127.X는 루프백 주소이기 때문에 포함하지 않으며, 0.0.0.0도 특수 주소로 포함하지 않는다.)

### 클래스 B
- 2^16-2 = 한 네트워크당 호스트 ID 약 6만 5천개
- 네트워크 주소 범위: 128~191로 시작

### 클래스 C
- 2^8-2 = 한 네트워크당 호스트 ID 254개
- 네트워크 주소 범위: 192~223으로 시작

- 주소 2개를 빼는 이유: 맨 앞자리는 네트워크 주소로 남겨두며 마지막 주소는 브로드캐스팅 주소로 남겨놓는다.
- 클래스풀의 문제점: 네트워크 크기가 작은 경우 큰 네트워크를 필요로 하는 조직은 여러개를 확보해야 하는 어려움이 생기며, 작은 네트워크가 필요한 조직의 경우 IP가 낭비된다.

## 4. 클래스리스와 서브넷마스크, 서브네팅
**클래스리스**: 클래스로 나누는 것이 아닌 서브넷마스크를 중심으로 어디까지가 네트워크 주소이고 어디까지가 호스트 주소인지를 나눈다.
- 서브네팅: 네트워크를 나눈다는 의미
- 서브넷: 서브 네트워크로, 쪼개진 네트워크를 의미
- 서브넷 마스크: 서브네트워크를 위한 비트마스크 (네트워크 주소 부분만 모두 1, 호스트 주소 부분은 모두 0으로 설정해서 나눈다.)

### 클래스풀과 클래스리스의 차이
**클래스풀 네트워킹(Classful Networking)**
- IP 주소를 A, B, C, D, E 클래스 기반으로 구분한다.
- 각 클래스는 고정된 서브넷 마스크(FSM)을 가지고 있다
  - A: 255.0.0.0
  - B: 255.255.0.0
  - C: 255.255.255.0
- 고정된 서브넷마스크를 사용해 네트워크와 호스트 부분을 나눈다.
- IP 주소의 첫번째 옥텟에 따라 클래스가 결정된다.

**클래스리스 네트워킹(Classless Inter-Domain Routing, CIDR)**
- 클래스 기반의 제한을 없앤다.
- 고정된 서브넷 마스크 대신 가변 길이 서브넷 마스크를 사용하여 네트워크와 호스트부분을 동적으로 나눈다.
- CIDR 표기법을 사용해 네트워크를 나눈다.
- 더 유연하게 네트워크를 할당하고, 주소 낭비를 줄일 수 있다.

## 5. 공인IP(public IP)와 사설IP(private IP)와 NAT
IP 주소의 부족을 공인 IP와 사설 IP로 나누고 중간에 NAT라는 기술을 통해 해결한다.
NAT(Network Address Transition)은 패킷이 트래픽 라우팅 장치를 통해 전송되는 동안 패킷의 IP 주소를 변경, IP 주소를 다른 IP 주소로 매핑하는 방법이다.
NAT를 통해 내부 네트워크 IP가 노출되지 않는다는 점 등이 장점이다.
