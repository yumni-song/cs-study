# 웹브라우저의 캐시

## 1. 로컬스토리지와 오리진의 개념
**로컬스토리지**: 웹 스토리지 객체로, 브라우저 내에 {key: value} 형태로 오리진에 종속되어 저장되는 데이터다.
  - 하나의 키에 하나의 값만 저장된다.
  - 데이터는 평생동안 로컬 저장소에 저장된다. 
  - 최대 저장용량은 5MB이다.
  - 로컬스토리지 데이터는 자동으로 서버로 전송되지 않는다. (쿠키는 자동 전송됨)
**오리진**: 같은 브라우저 내에서 공유되며, protocal, hostname, port를 포함한다.

## 2. 세션스토리지
**세션스토리지**: 웹 스토리지 객체로, 브라우저 내에 {key: value} 형태로 오리진에 종속되어 저장되는 데이터다. 다만, 동일한 오리진이어도 브라우저의 각 탭마다 **독립적**으로 저장된다. 즉, 다른 탭에서 세션스토리지에 저장된 데이터에 **접근할 수 없다.**
  - 하나의 키에 하나의 값만 저장된다.
  - 최대 저장용량으 5MB이다.
  - 사용자가 브라우저에서 탭을 닫으면 데이터는 만료된다.
  - 보통 세션스토리지보다는 로컬스토리지를 많이 쓴다.

## 3. 쿠키(Cookie)
**쿠키**: 브라우저에 저장된 데이터 조각
  - 서버에서 쿠키를 먼저 만드는게 일반적
  - 서버에서 응답해더로 Set-Cookie로 설정해 쿠키를 보내면 클라이언트에서 요청헤더 Cookie에 설정되어 자동으로 서버에 전달되고 브라우저에 저장된다. 
  - 쿠키는 클라이언트와 서버 둘 다 조작이 가능하지만 보통 서버에서 만료기한 등을 설정한다. 
  - 저장용량은 최대 4kb이다.
  - 쿠키에는 민감한 정보가 담길 수 있으므로 보통 제어권을 클라이언트에 두지 않는다.
**세션 쿠키**: Expires 또는 Max-Age 속성을 지정하지 않은 것으로, 브라우저가 종료되면 쿠키도 사라진다.
**영구 쿠기**: Expires 또는 Max-Age 속성을 지정해서 특정 날짜 또는 일정 기간이 지나면 삭제되게 만든 것으로, 브라우저를 닫을 때도 만료되지 않는다.
**문법**
  - secure: https로만 쿠키를 주고받을 수 있게 하는 옵션이다.
  - httponly: 공격자가 쿠키를 자바스크립트로 빼낼 수 없게 만든다.
  - samesite: 요청이 동일한 도메인에서 시작된 경우에만 쿠키가 애플리케이션으로 전송되도록 허용한다.


## 6. 로컬스토리지, 세션스토리지, 무미의 공통점과 차이점
**공통점**
1. 서버에 대한 요청을 줄여 서버부하를 방지할 수 있다.
2. 캐싱으로 인해 다운로드하는 컨텐츠가 줄어들어 웹사이트의 컨텐츠를 더 빨리 다운로드가 가능하다.
3. 사이트 기본 설정 커스터마이징을 저장하거나 로그인 상태를 유지할 때 사용될 수 있다.

**차이점**
| 구분 | 쿠키 (Cookie) | 로컬 스토리지 (Local Storage) | 세션 스토리지 (Session Storage) |
|------|--------------|-----------------------------|---------------------------------|
| 최대 저장 용량 | 4KB | 5MB | 5MB |
| 브라우저 허용 | HTML4 + HTML5 | HTML5 | HTML5 |
| 접근 범위 | 오리진 (Origin) | 오리진 (Origin) | 탭 (Tab) |
| 만료 기한 | 수동 설정 | 영구적 | 탭 닫으면 소멸 |
| 설정 주체 | 클라이언트 + 서버 | 클라이언트 | 클라이언트 |
| 요청 시 서버 자동 전송 | O | X | X |
